Security Bulletin

Microsoft Security Bulletin MS06-075 - Důležité
===============================================

Chyba zabezpečení systému Windows umožňuje zvýšení úrovně oprávnění (926255)
----------------------------------------------------------------------------

Publikováno: 12. prosince 2006

**Verze** 1.0

#### Shrnutí

**Komu je tento dokument určen:** Zákazníkům, kteří používají systém Microsoft Windows

**Dopad této chyby zabezpečení:** Zvýšení úrovně oprávnění

**Maximální stupeň závažnosti:** Vysoký

**Doporučení:** Zákazníci by měli tuto aktualizaci nainstalovat co možná nejdříve.

**Nahrazené aktualizace zabezpečení:** Žádná.

**Upozornění:** Žádná.

**Testovaný software a umístění pro stahování aktualizace zabezpečení:**

**Software obsahující tuto chybu:**

Podrobné informace o chybě zabezpečení  
--------------------------------------
  
#### Chyba zabezpečení týkající se poškození souboru manifestu – CVE-2006-5585:
  
Byla zjištěna chyba zabezpečení umožňující [zvýšení úrovně oprávnění](http://go.microsoft.com/fwlink/?linkid=21142) ve způsobu, jakým systém Microsoft Windows spouští aplikace se speciálně vytvořenými soubory manifestů. Tato chyba zabezpečení by mohla přihlášenému uživateli umožnit získat úplnou kontrolu nad systémem.
  
#### Skutečnosti snižující závažnost chyby zabezpečení týkající se poškození souboru manifestu – CVE-2006-5585:
  
-   Tuto chybu může zneužít pouze útočník s platnými pověřeními pro přihlášení, která by mu umožňovala se místně přihlásit. Tuto chybu zabezpečení nemohou zneužít vzdálení ani anonymní uživatelé.
  
#### Možná zástupná řešení chyby zabezpečení týkající se poškození souboru manifestu – CVE-2006-5585:
  
Pro tuto chybu zabezpečení nebylo nalezeno žádné zástupné řešení.
  
#### Nejčastější dotazy týkající se chyby zabezpečení související s poškozením souboru manifestu – CVE-2006-5585:
  
**Jaký je rozsah této chyby zabezpečení?**  
Jedná se o chybu zabezpečení, která by mohla vést ke [zvýšení úrovně oprávnění](http://go.microsoft.com/fwlink/?linkid=21142). Útočník, který by tuto chybu zabezpečení zneužil, by mohl získat úplnou kontrolu nad postiženým systémem. Útočník by tak mohl instalovat programy, zobrazovat, měnit či odstraňovat data nebo vytvářet nové účty s úplnými uživatelskými právy. Aby se mohl útočník pokusit tuto chybu zneužít, musel by být schopen místně se přihlásit k systému a spustit program.
  
**Co tuto chybu zabezpečení způsobuje?**  
Chybné zpracování a správa souborů manifestů podsystémem CSRSS (Client Server Run-Time Subsystem).
  
**Co je podsystém CSRSS (Client Server Run-Time Subsystem)?**  
**Csrss** je uživatelská část podsystému Win32 (Win32.sys je část jádra). Jedná se o zkratku anglického *client/server run-time subsystem*, což je nezbytný podsystém, který musí být stále spuštěný. Zodpovídá za konzoly systému Windows a slouží k vytváření a odstraňování podprocesů.
  
**K čemu by mohl útočník tuto chybu zabezpečení zneužít?**  
Útočník, který by tuto chybu zabezpečení zneužil, by mohl získat úplnou kontrolu nad postiženým systémem.
  
**Kdo by mohl tuto chybu zabezpečení zneužít?**  
Aby mohl útočník tuto chybu zneužít, musel by být schopen místně se přihlásit k systému a spustit program.
  
**Jak by mohl útočník tuto chybu zabezpečení zneužít?**  
Aby mohl útočník tuto chybu zneužít, musel by být nejdříve schopen přihlásit se k systému. Útočník by mohl následně spustit speciálně vytvořený program, pomocí kterého by mohl zneužít tuto chybu zabezpečení, a získat tak úplnou kontrolu nad postiženým systémem.
  
**Které systémy jsou touto chybou ohroženy nejvíce?**  
Chybou jsou ohroženy především pracovní stanice a terminálové servery. Servery by byly více ohroženy v případě, že by správci umožnili uživatelům přihlašovat se k nim a spouštět programy. V praxi se nicméně osvědčilo toto v žádném případě uživatelům neumožňovat.
  
**Bylo by možné tuto chybu zabezpečení zneužít prostřednictvím Internetu?**  
Ne. Útočník se musí přihlásit ke konkrétnímu systému, aby jej mohl napadnout. Útočník nemůže pomocí této chyby zabezpečení program zavést a spustit prostřednictvím vzdáleného přístupu.
  
**Co je podstatou této aktualizace?**  
Tato aktualizace odstraní tuto chybu zabezpečení tím, že upraví způsob, kterým podsystém CSRSS (Client Server Run-time Subsystem) ověřuje vložené soubory manifestů před předáním dat do přidělené vyrovnávací paměti.
  
**Byla v době zveřejnění tohoto bulletinu tato chyba zabezpečení veřejně známá?**  
Ne, tento problém nebyl dříve veřejně známou chybou zabezpečení.
  
**Jak tato chyba zabezpečení souvisí s článkem 921337 znalostní báze Microsoft Knowledge Base**  
[Článek 921337 znalostní báze Microsoft Knowledge Base](http://support.microsoft.com/kb/921337/en-us) se zabývá podobným problémem s manifestem souborů, který může způsobit restartování systému. Při revizi tohoto problému zjistila společnost Microsoft potenciální chybu zabezpečení, která se odstraňuje vydáním této aktualizace zabezpečení.

Tyto nástroje nemusí některé softwarové aktualizace zjistit. V takovém případě mohou správci použít funkce serveru SMS a nasměrovat aktualizace do určitých systémů.
