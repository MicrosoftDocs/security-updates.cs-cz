Security Bulletin

Microsoft Security Bulletin MS06-029 - Kritické
===============================================

Chyba zabezpečení serveru Microsoft Exchange Server se spuštěnou aplikací Outlook Web Access umožňuje vkládání skriptu (912442)
-------------------------------------------------------------------------------------------------------------------------------

Publikováno: 13. června 2006

**Verze:** 1.0

#### Shrnutí

**Komu je tento dokument určen:** Zákazníkům, kteří používají Microsoft Exchange Server se spuštěnou aplikací Microsoft Outlook Web Access

**Dopad této chyby zabezpečení:** Vzdálené spuštění kódu

**Stupeň maximální závažnosti:** Vysoký

**Doporučení:** Zákazníci by měli uvažovat o instalaci této aktualizace zabezpečení.

**Nahrazené aktualizace zabezpečení:** Žádné

**Upozornění:** [Článek 912442 znalostní báze Microsoft Knowledge Base](http://support.microsoft.com/kb/912442) popisuje aktuálně známé problémy, s nimiž se mohou zákazníci setkat v případě instalace této aktualizace zabezpečení. V článku jsou také uvedena doporučená řešení těchto problémů. Další informace získáte [v článku 912442 znalostní báze Microsoft Knowledge Base](http://support.microsoft.com/kb/912442).

**Testovaný software a umístění pro stahování aktualizace zabezpečení:**

**Software obsahující tuto chybu:**

-   Microsoft Exchange 2000 Server Pack 3 s kumulativní aktualizací Exchange 2000 Server Post-Service Pack 3 ze srpna 2004 – [stáhnout aktualizaci (KB912442)](http://www.microsoft.com/downloads/details.aspx?familyid=746ce64e-3186-422b-a13b-004e7942189b)

-   Microsoft Exchange Server 2003 Service Pack 2 – [stáhnout aktualizaci (KB912442)](http://www.microsoft.com/downloads/details.aspx?familyid=http://www.microsoft.com/downloads/details.aspx?familyid=c777bc9f-52b7-4f17-96c7-daf3b9987d70)

Uvedený software byl testován, zda neobsahuje tuto chybu. Pro ostatní verze buď již nejsou poskytovány opravy zabezpečení, nebo tuto chybu neobsahují. Podrobné informace o chybě zabezpečení
--------------------------------------

<span></span>
#### Chyba zabezpečení serveru Microsoft Exchange Server se spuštěnou aplikací Outlook Web Access – CVE- CVE-2006-1193

Na serveru Exchange Server se spuštěnou aplikací Outlook Web Access (OWA) existuje chyba zabezpečení týkající se vkládání skriptu. Útočník by mohl tuto chybu zabezpečení zneužít sestavením e-mailové zprávy se speciálně vytvořeným skriptem. V případě spuštění tohoto skriptu by mohlo dojít k jeho spuštění v kontextu zabezpečení uživatele na straně klienta. Při pokusech o zneužití této chyby zabezpečení je nutná interakce s uživatelem.

#### Skutečnosti snižující závažnost rizika chyby zabezpečení serveru Microsoft Exchange Server se spuštěnou aplikací Outlook Web Access – CVE- CVE-2006-1193:

-   Aby tato chyba zabezpečení mohla být zneužita, musel by uživatel přečíst speciálně vytvořenou e-mailovou zprávu v aplikaci Outlook Web Access.

#### Zástupná řešení chyby zabezpečení serveru Microsoft Exchange Server se spuštěnou aplikací Outlook Web Access – CVE- CVE-2006-1193:

Společnost Microsoft testovala následující zástupná řešení. Přestože tato zástupná řešení neopravují samotnou chybu zabezpečení, pomáhají blokovat známé způsoby útoku. Případná omezení funkčnosti způsobená daným zástupným řešením jsou uvedena v následující části.

-   **Zakázání aplikace Outlook Web Access (OWA) v počítači se spuštěným serverem Exchange Server**
    Zakázání aplikace Outlook Web Access pomáhá chránit postižený systém před pokusy o zneužití této chyby zabezpečení. Chcete-li zakázat aplikaci Outlook Web Access, postupujte podle následujících kroků:
    1.  Klepněte na tlačítko **Start**, přejděte na příkaz **All Programs (Všechny programy)**, přejděte na položku **Microsoft Exchange** a klepněte na příkaz **System Manager (Správce systému)**.
    2.  Rozbalte položku **Servers (Servery),** potom položku **Server,** pak rozbalte položku **Protocols (Protokoly)** a nakonec položku **HTTP**.
    3.  Klepněte pravým tlačítkem myši na položku **Exchange Virtual Server (Virtuální server Exchange)** a potom klepněte na příkaz **Stop (Zastavit)**.

**Poznámka:** Nad ikonou serveru Exchange Virtual Server (Virtuální server Exchange) se zobrazí červený křížek označující, že server byl zastaven. Pokud se nyní uživatelé pokusí pracovat s e-maily v aplikaci OWA, zobrazí se chybová zpráva **Stránku nelze zobrazit**.

**Dopad tohoto zástupného řešení:** Toto zástupné řešení brání uživatelům pracovat s poštovní schránkou prostřednictvím aplikací Outlook Web Access (OWA), Outlook Mobile Access (OMA) a Exchange Server ActiveSync.

#### Nejčastější dotazy týkající se chyby zabezpečení serveru Microsoft Exchange Server se spuštěnou aplikací Outlook Web Access – CVE-2006-1193:

**Jaký je rozsah této chyby zabezpečení?**
Byla zjištěna chyba zabezpečení týkající se vkládání skriptu, která by útočníkovi mohla umožnit spuštění škodlivého skriptu. Tento škodlivý skript by byl spuštěn v kontextu zabezpečení daného uživatele na straně klienta. Tento skript by mohl v uživatelově počítači provést jakoukoli akci, kterou může web provádět. Mohlo by se jednat o sledování webové relace a předávání informací třetí straně, spuštění dalšího kódu v systému uživatele a čtení nebo zapisování souborů cookie.

**Co je aplikace Outlook Web Access?**
Microsoft Outlook Web Access (OWA) je služba serveru Exchange Server. Při použití aplikace Outlook Web Access může server se softwarem Exchange Server fungovat také jako web, který oprávněným uživatelům umožňuje číst nebo odesílat e-maily, spravovat kalendář a provádět jiné akce s e-maily prostřednictvím Internetu.

**Co tuto chybu zabezpečení způsobuje?**
Tato chyba zabezpečení je způsobena skutečností, že aplikace Outlook Web Access za určitých okolností nesprávně filtruje skripty v e-mailové zprávě.

**Jak by mohl útočník tuto chybu zabezpečení zneužít?**
Útočník by se mohl pokusit zneužít tuto chybu zabezpečení tak, že uživateli odešle speciálně vytvořenou zprávu. Uživatel by potom musel tuto zprávu otevřít pomocí aplikace Outlook Web Access. Zpráva by mohla následně v postiženém systému způsobit spuštění skriptu v kontextu relace uživatele aplikace Outlook Web Access.

**Které systémy jsou touto chybou ohroženy nejvíce?**
Nejvíce jsou ohroženy servery Microsoft Exchange s povoleným používáním aplikace Outlook Web Access.

**Co je podstatou této aktualizace?**
Aktualizace odstraňuje tuto chybu zabezpečení tím, že změní způsob, jakým aplikace Outlook Web Access provádí analýzu paketů protokolu HTML.

**Byla v době zveřejnění tohoto bulletinu tato chyba zabezpečení veřejně známá?**
Ne. Společnost Microsoft byla o této chybě informována důvěryhodným zdrojem. Společnost Microsoft nemá žádné informace, které by naznačovaly, že tato chyba zabezpečení byla v době vydání tohoto bulletinu veřejně známá.

**Měla v době zveřejnění tohoto bulletinu společnost Microsoft nějaké informace o tom, že tato chyba zabezpečení byla zneužita?**
Ne. Společnost Microsoft neměla žádné informace, které by naznačovaly, že daná chyba zabezpečení byla v době původního zveřejnění tohoto bulletinu zneužita k útoku na zákazníky, a nezaznamenala ani žádné důkazy nebo publikované příklady kódu, které by naznačovaly, že tato chyba byla zneužita.

Další informace o tomto postupu najdete na následujícím [webu](http://go.microsoft.com/fwlink/?linkid=33341). Některé aktualizace zabezpečení vyžadují po restartování počítače oprávnění správce. Správci mohou k instalaci těchto aktualizací použít nástroj Elevated Rights Deployment Tool (k dispozici v sadě [SMS 2003 Administration Feature Pack](http://go.microsoft.com/fwlink/?linkid=33387) a [SMS 2.0 Administration Feature Pack](http://go.microsoft.com/fwlink/?linkid=21161)).
