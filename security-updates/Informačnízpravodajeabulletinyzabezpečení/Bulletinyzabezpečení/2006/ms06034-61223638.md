Security Bulletin

Microsoft Security Bulletin MS06-034 - Kritické
===============================================

Chyba zabezpečení ve službě Microsoft Internet Information Services (IIS) používající stránky ASP (Active Server Pages) umožňuje vzdálené spuštění kódu (917537)
----------------------------------------------------------------------------------------------------------------------------------------------------------------

Publikováno: 11. července 2006 | Aktualizováno: 12. července 2006

**Verze:** 1.1

**Stupeň maximální závažnosti:** Vysoký

Podrobné informace o chybě zabezpečení

<span></span>
#### Chyba zabezpečení služby Internet Information Services používající chybně formátované stránky ASP – CVE-2006-0026

Byla zjištěna chyba zabezpečení služby Internet Information Services (IIS) umožňující vzdálené spuštění kódu. Útočník by mohl tuto chybu zabezpečení zneužít sestavením speciálně vytvořeného souboru stránek ASP (Active Server Pages), což by mohlo potenciálně umožnit vzdálené spuštění kódu v případě, že služba IIS zpracuje speciálně vytvořený soubor. Útočník, který by tuto chybu zabezpečení zneužil, by mohl získat úplnou kontrolu nad postiženým systémem.

#### Skutečnosti snižující závažnost chyby zabezpečení služby Internet Information Services používající chybně formátované stránky ASP – CVE-2006-0026:

-   Ve službách IIS 5.0 a IIS 5.1 jsou aplikace s povolenými stránkami ASP ve výchozím nastavení spuštěny v aplikaci Pooled out of Process (Spouštěné mimo proces), což znamená, že jsou spuštěny v souboru DLLHOST.exe, který běží v kontextu účtu IWAM\_&lt;název\_počítače&gt; s nízkými oprávněními.
-   Ve výchozím nastavení nejsou služby IIS 5.1 v systému Windows XP Professional a IIS 6.0 v systému Windows Server 2003 povoleny.
-   Ve službě IIS 6.0 nejsou ve výchozím nastavení stránky ASP povoleny. Pokud povoleny jsou, jsou spuštěny v kontextu dílčího procesu W3WP.exe spuštěného jako účet NetworkService s nízkým oprávněním.
-   Tuto chybu by mohl zneužít pouze útočník s platnými pověřeními k přihlášení. Pokud však byl server záměrně konfigurován tak, aby umožňoval uživatelům, ať již anonymním nebo ověřeným, odesílat webový obsah, například stránky ASP, na weby, mohl by být server pomocí této chyby napaden.

#### Zástupná řešení chyby zabezpečení služby Internet Information Services používající chybně formátované stránky ASP – CVE-2006-0026:

Zatím nebylo nalezeno žádné zástupné řešení této chyby zabezpečení.

#### Nejčastější dotazy týkající se chyby zabezpečení služby Internet Information Services používající chybně formátované stránky ASP – CVE-2006-0026:

**Jaký je rozsah této chyby zabezpečení?**
Byla zjištěna chyba zabezpečení služby Internet Information Services (IIS) umožňující vzdálené spuštění kódu. Důvodem je způsob zpracování stránek ASP (Active Server Pages) službou IIS. Útočník by mohl tuto chybu zabezpečení zneužít sestavením speciálně vytvořeného souboru stránek ASP (Active Server Pages), což by mohlo povolit vzdálené spuštění kódu v případě, že služba IIS zpracuje tento soubor. Útočník, který by tuto chybu zabezpečení zneužil, by mohl získat úplnou kontrolu nad postiženým systémem.

**Co tuto chybu zabezpečení způsobuje?**
Nekontrolovaná vyrovnávací paměť ve službě IIS.

**Co jsou stránky ASP (Active Server Pages)?**
Microsoft ASP (Active Server Pages) je technologie skriptování na straně serveru, kterou lze použít k vytváření dynamických a interaktivních webových aplikací. Stránka ASP je stránka HTML obsahující skripty na straně serveru, které jsou před odesláním do prohlížeče uživatele zpracovány webovým serverem.

**K čemu by mohl útočník tuto chybu zabezpečení zneužít?**
Útočník, který by tuto chybu zabezpečení zneužil, by mohl získat úplnou kontrolu nad postiženým systémem.

**Kdo by mohl tuto chybu zabezpečení zneužít?**
Tuto chybu by mohl zneužít pouze útočník s platnými pověřeními k přihlášení k serveru. Pokud však byl server záměrně konfigurován tak, aby umožňoval uživatelům, ať již anonymním nebo ověřeným, odesílat webový obsah, například stránky ASP, na weby, mohl by být server zneužitím této chyby napaden.

**Jak by mohl útočník tuto chybu zabezpečení zneužít?**
Útočník by se mohl pokusit tuto chybu zabezpečení zneužít vytvořením speciálně vytvořeného souboru ASP, který by odeslal do postiženého systému. Pokud by služba IIS soubor zpracovala, mohlo by dojít ke spuštění kódu postiženým systémem.

**Které systémy jsou touto chybou ohroženy nejvíce?**
Touto chybou zabezpečení jsou ohroženy především systémy Windows 2000 Service Pack 4 se spuštěnou službou IIS 5.0, protože služba IIS je ve výchozím nastavení povolena. Systémy Windows XP Professional a Windows Server 2003 jsou ohroženy také v případě povolení služby.

**Bylo by možné tuto chybu zabezpečení zneužít prostřednictvím Internetu?**
Ano. Útočník by se mohl pokusit o zneužití této chyby zabezpečení prostřednictvím Internetu, pokud má přístup k zápisu na server. Doporučené postupy pro používání brány firewall a standardní výchozí konfigurace této brány zajišťují ochranu před útoky pocházejícími z Internetu. Společnost Microsoft zveřejnila informace o tom, jak můžete přispět k zabezpečení svého počítače. Koncoví uživatelé mohou navštívit web [Chraňte svůj počítač](http://www.microsoft.com/cze/athome/security/protect/windowsxpsp2/default.mspx). Odborníci v oblasti IT mohou navštívit web [Centrum doporučených postupů zabezpečení](http://www.microsoft.com/cze/security/guidance/default.mspx).

**Byla v době zveřejnění tohoto bulletinu tato chyba zabezpečení veřejně známá?**

**Měla v době zveřejnění tohoto bulletinu společnost Microsoft nějaké informace o tom, že tato chyba zabezpečení byla zneužita?**
Ne. Společnost Microsoft neměla žádné informace, které by naznačovaly, že daná chyba zabezpečení byla v době původního zveřejnění tohoto bulletinu zneužita k útoku na zákazníky, a nezaznamenala ani žádné důkazy nebo publikované příklady kódu, které by naznačovaly, že tato chyba byla zneužita.

**Požadavek na restartování**

Tato aktualizace nevyžaduje restartování. Instalační služba pozastaví příslušnou službu, nainstaluje aktualizaci a pak danou službu opět spustí. Pokud však z nějakého důvodu nebude možné službu zastavit nebo pokud budou některé potřebné soubory používány, bude nutné počítač restartovat. Jestliže k tomuto chování dojde, zobrazí se zpráva doporučující restartování počítače.

Pokud nechcete počítač restartovat, před instalací aktualizace zabezpečení zastavte všechny služby obsahující uvedenou chybu a ukončete všechny aplikace, které by mohly používat soubory obsahující uvedenou chybu. Další informace o důvodech, které mohou způsobit zobrazení výzvy k restartování počítače, najdete v [článku 887012 znalostní báze Microsoft Knowledge Base](http://support.microsoft.com/kb/887012).

Tato aktualizace zabezpečení podporuje technologii HotPatching v systémech, které používají verzi souboru Asp.dll 6.0.3790.1830. Pokud používáte technologii HotPatching, můžete instalovat aktualizace zabezpečení v systémech se spuštěným operačním systémem Microsoft Windows Server 2003 Service Pack 1, aniž by bylo nutné restartovat servery. Další informace o technologii HotPatching a instalaci této aktualizace zabezpečení pomocí technologie HotPatching získáte v [článku 897341 znalostní báze Microsoft Knowledge Base](http://support.microsoft.com/kb/897341).

Tato aktualizace zabezpečení, která je k dispozici prostřednictvím webu Windows Update, Microsoft Update a funkce Automatické aktualizace, využívá technologii Hotpatching a nevyžaduje restartování serveru

**Požadavek na restartování**

Tato aktualizace nevyžaduje restartování. Instalační služba pozastaví příslušnou službu, nainstaluje aktualizaci a pak danou službu opět spustí. Pokud však z nějakého důvodu nebude možné službu zastavit nebo pokud budou některé potřebné soubory používány, bude nutné počítač restartovat. Jestliže k tomuto chování dojde, zobrazí se zpráva doporučující restartování počítače.

Pokud nechcete počítač restartovat, před instalací aktualizace zabezpečení zastavte všechny služby obsahující uvedenou chybu a ukončete všechny aplikace, které by mohly používat soubory obsahující uvedenou chybu. Další informace o důvodech, které mohou způsobit zobrazení výzvy k restartování počítače, najdete v [článku 887012 znalostní báze Microsoft Knowledge Base](http://support.microsoft.com/kb/887012).

    **Poznámka:** Vzhledem k různým verzím systému Microsoft Windows se následující postup může lišit od postupu ve vašem počítači. Pokud tomu tak je, bude třeba před dokončením následujících kroků konzultovat dokumentaci k produktu.

    1.  Klepněte na tlačítko **Start** a potom na příkaz **Hledat**.
    2.  Ve skupinovém rámečku **Průvodce vyhledáváním** v podokně **Výsledky hledání** klepněte na odkaz **Všechny soubory a složky**.
    3.  Do pole **Část nebo celý název souboru** zadejte název souboru z příslušné tabulky a klepněte na tlačítko **Hledat**.
    4.  V seznamu souborů klepněte pravým tlačítkem myši na název souboru z příslušné tabulky s informacemi o souborech a klepněte na příkaz **Vlastnosti**.

        **Poznámka:** Atributy (kromě verze souboru) se mohou při instalaci změnit. Umožňuje správcům identifikovat počítače se systémem Windows, které vyžadují aktualizace zabezpečení, a provést řízenou instalaci těchto aktualizací v celé rozlehlé síti s minimálním dopadem na koncové uživatele. Další informace o tom, jak mohou správci pomocí serveru SMS 2003 instalovat aktualizace zabezpečení, získáte na webu [SMS 2003 Security Patc](http://go.microsoft.com/fwlink/?linkid=22939)h Management. Další informace o serveru SMS získáte na [webu serveru SMS](http://go.microsoft.com/fwlink/?linkid=21158).

**Poznámka:** Server SMS využívá nástroj MBSA (Microsoft Baseline Security Analyzer), nástroj pro rozpoznávání sady Microsoft Office a nástroj pro vyhledávání aktualizací v rozlehlých sítích (Enterprise Update Scanning Tool – EST), a poskytuje tak podporu pro zjišťování a instalaci aktualizací uvedených v bulletinech zabezpečení. Tyto nástroje nemusí některé softwarové aktualizace zjistit. V takovém případě mohou správci použít funkce serveru SMS a nasměrovat aktualizace do určitých systémů. Další informace o tomto postupu najdete na následujícím [webu](http://go.microsoft.com/fwlink/?linkid=33341). Některé aktualizace zabezpečení vyžadují po restartování počítače oprávnění správce. Správci mohou k instalaci těchto aktualizací použít nástroj Elevated Rights Deployment Tool (k dispozici v sadě [SMS 2003 Administration Feature Pack](http://go.microsoft.com/fwlink/?linkid=33387) a [SMS 2.0 Administration Feature Pack](http://go.microsoft.com/fwlink/?linkid=21161)).
