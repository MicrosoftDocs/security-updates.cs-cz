Security Bulletin

Microsoft Security Bulletin MS06-036 - Kritické
===============================================

------------------------------------------------------------------------------

Publikováno: 11. července 2006

**Verze:** 1.0

**Stupeň maximální závažnosti:** Kritický

**Nahrazené aktualizace zabezpečení:** Žádné

Podrobné informace o chybě zabezpečení
--------------------------------------

<span></span>
#### Chyba zabezpečení týkající se přetečení vyrovnávací paměti ve službě klienta DHCP – CVE-2006-2372

Byla zjištěna chyba zabezpečení služby klienta DHCP, která může způsobit vzdálené spuštění kódu, čímž by mohla útočníkovi, který tuto chybu zabezpečení úspěšně zneužije, umožnit převzetí úplné kontroly nad postiženým systémem.

#### Skutečnosti snižující závažnost rizika spojeného s chybou zabezpečení týkající se přetečení vyrovnávací paměti ve službě klienta DHCP – CVE-2006-2372:

-   Útok může být úspěšný pouze v případě, že útočník pošle postiženému hostiteli chybně formátovanou komunikaci DHCP ze stejné podsítě.

#### Možná zástupná řešení chyby zabezpečení týkající se přetečení vyrovnávací paměti ve službě klienta DHCP – CVE-2006-2372:

Společnost Microsoft testovala následující zástupná řešení. Přestože tato zástupná řešení neopravují samotnou chybu zabezpečení, pomáhají blokovat známé způsoby útoku. Případná omezení funkčnosti způsobená daným zástupným řešením jsou uvedena v následující části.

-   **Použití statické adresy IP**

    Konfiguraci systému pro použití statické adresy IP proveďte následujícím způsobem:

    1.  Vyžádejte si svou statickou adresu IP od správce sítě.
    2.  Klepněte na tlačítko **Start**, pak klepněte na příkaz **Ovládací panely** a potom na položku **Připojení k síti a Internetu**.
    3.  Pravým tlačítkem klepněte na připojení, pro které chcete použít statickou adresu IP, a pak klepněte na příkaz **Vlastnosti**.
    4.  Vyberte položku **Protokol sítě Internet (TCP/IP)** a klepněte na příkazy **Vlastnosti**.
    5.  Na kartě **Obecné** klepněte na přepínač **Použít následující adresu IP** a **Použít následující adresy serverů DNS**.
    6.  Zadejte svou statickou adresu IP, masku podsítě, výchozí bránu a servery DNS.
    7.  Poklepejte na tlačítko **OK**.
    8.  Opakujte kroky 3 až 7 pro každý síťový adaptér.

-   **Zakázání služby klienta DHCP**

    Zakázáním služby klienta DHCP přispějete ke zvýšení ochrany postiženého systému před pokusy o zneužití této chyby zabezpečení. Chcete-li zakázat službu klienta DHCP, postupujte podle následujících kroků:

    1.  Klepněte na tlačítko **Start** a potom na příkaz **Ovládací panely**. Můžete také přejít na příkaz **Nastavení** a pak klepnout na příkaz **Ovládací panely**.
    2.  Poklepejte na ikonu **Nástroje pro správu**.
    3.  Poklepejte na položku **Služby**.
    4.  Poklepejte na položku **Klient DHCP**.
    5.  V seznamu **Typ spouštění** klepněte na položku **Zakázáno**.
    6.  Klepněte na tlačítko **Zastavit** a pak klepněte na tlačítko **OK**.

    Službu DHCP je také možné zastavit a zakázat zadáním následujícího příkazu na příkazovém řádku:

    `sc stop DHCP & sc config DHCP start= disabled`

    **Dopad tohoto zástupného řešení:** Pokud zakážete službu DHCP, nebude možné automaticky načíst adresu IP a připojit se s její pomocí k síti.

#### Nejčastější dotazy k chybě zabezpečení týkající se přetečení vyrovnávací paměti ve službě klienta DHCP – CVE-2006-2372:

**Jaký je rozsah této chyby zabezpečení?**
Jedná se o chybu zabezpečení služby klienta DHCP umožňující vzdálené spuštění kódu. Útočník, který by tuto chybu zabezpečení úspěšně zneužil, by mohl vzdáleně získat úplnou kontrolu nad postiženým systémem. Útočník by tak mohl instalovat programy, zobrazovat, měnit či odstraňovat data nebo vytvářet nové účty s úplnými uživatelskými právy.

**Co tuto chybu zabezpečení způsobuje?**
Nekontrolovaná vyrovnávací paměť v rámci služby klienta DHCP.

**Co je DHCP?**
Protokol DHCP (Dynamic Host Configuration Protocol) je standard protokolu sítě Internet, jehož cílem je zjednodušit správu síťových adres, a to tak, že adresy IP a související konfigurační data budou centrálně spravovány serverem. Windows Server poskytuje službu DHCP Server, která umožňuje počítači se serverem pracovat jako server DHCP a poskytovat síti nastavení konfigurace pro klienty s povolenou službou DHCP, jak je popsáno v dokumentu DHCP IETF [RFC 2131](http://www.faqs.org/rfcs/rfc2131.html).

**K čemu by mohl útočník tuto chybu zabezpečení zneužít?**
Útočník, který by tuto chybu zabezpečení zneužil, by mohl získat úplnou kontrolu nad postiženým systémem.

**Kdo by mohl tuto chybu zabezpečení zneužít?**
Anonymní uživatel by mohl zneužít tuto chybu zabezpečení odesláním chybně formátované komunikace DHCP postiženému klientovi ve stejném segmentu sítě. V obvyklé výchozí síťové konfiguraci, při níž není zapnuto předávání DHCP nebo BOOTP, je útok omezen na místní podsíť.

**Jak by mohl útočník tuto chybu zabezpečení zneužít?**
Útočník by mohl zneužít tuto chybu zabezpečení odesláním odpovědi na požadavek DHCP klienta v místní podsíti prostřednictvím chybně formátovaných paketů.

**Které systémy jsou touto chybou ohroženy nejvíce?**
Touto chybou jsou ohroženy zejména systémy Microsoft Windows 2000, Windows XP a Windows Server 2003.

**Je postižení systémů Windows 98, Windows 98 Druhé vydání nebo Windows Millennium Edition touto chybou kritické?**
Ne. Systémy Windows 98, Windows 98 Druhé vydání a Windows Millennium Edition neobsahují součást obsahující tuto chybu.

**Bylo by možné tuto chybu zabezpečení zneužít prostřednictvím Internetu?**
Útočník by se mohl pokusit o zneužití této chyby zabezpečení prostřednictvím Internetu. Doporučené postupy pro používání brány firewall a standardní výchozí konfigurace této brány zajišťují ochranu před útoky pocházejícími z Internetu. Společnost Microsoft zveřejnila informace o tom, jak můžete přispět k zabezpečení svého počítače. Koncoví uživatelé mohou navštívit web [Chraňte svůj počítač](http://www.microsoft.com/cze/security/protect/). Odborníci v oblasti IT mohou navštívit web [Centrum doporučených postupů zabezpečení](http://www.microsoft.com/cze/security/guidance/).

**Co je podstatou této aktualizace?**
Tato aktualizace odstraní tuto chybu zabezpečení tím, že ověří způsob, jakým klient DHCP zpracovává komunikace související s protokolem DHCP.

**Byla v době zveřejnění tohoto bulletinu tato chyba zabezpečení veřejně známá?**
Ne. Společnost Microsoft byla o této chybě informována důvěryhodným zdrojem. Společnost Microsoft nemá žádné informace, které by naznačovaly, že tato chyba zabezpečení byla v době vydání tohoto bulletinu veřejně známá. **Informace o instalaci**

Pokud chcete tuto opravu zabezpečení nainstalovat bez zásahu uživatele, zadejte na příkazovém řádku systému Microsoft Windows XP následující příkaz:

**Windowsxp- kb914388-x86-enu /quiet**

**Poznámka:** Použitím přepínače **/quiet** se potlačí všechny zprávy. Potlačí se také zprávy o selhání. Správci by měli při použití přepínače **/quiet** používat jednu z podporovaných metod ověření, zda instalace proběhla úspěšně. Správci by také měli zkontrolovat, zda soubor protokolu KB914388.log neobsahuje při používání tohoto přepínače zprávy o selhání.

Jestliže chcete tuto opravu zabezpečení nainstalovat bez vynucení restartování, zadejte na příkazovém řádku systému Windows XP následující příkaz:

**Windowsxp- kb914388-x86-enu /norestart**

Informace o způsobu instalace této aktualizace zabezpečení pomocí služby SUS získáte na [webu služ](http://go.microsoft.com/fwlink/?linkid=21125)by Software Update Services. Další informace o způsobu instalace této aktualizace zabezpečení pomocí služby Windows Server Update Services získáte na [webu služby Windows Server Update Services](http://go.microsoft.com/fwlink/?linkid=50120). Tato aktualizace zabezpečení bude také k dispozici prostřednictvím [webu Microsoft Update](http://update.microsoft.com/microsoftupdate).

**Požadavek na restartování**

Po instalaci této aktualizace zabezpečení je třeba restartovat počítač. Pokud nechcete počítač restartovat, před instalací aktualizace zabezpečení zastavte všechny služby obsahující uvedenou chybu a ukončete všechny aplikace, které by mohly používat soubory obsahující uvedenou chybu. Další informace o důvodech, které mohou způsobit zobrazení výzvy k restartování počítače, najdete v [článku 887012 znalostní báze Microsoft Knowledge Base](http://support.microsoft.com/kb/887012).
