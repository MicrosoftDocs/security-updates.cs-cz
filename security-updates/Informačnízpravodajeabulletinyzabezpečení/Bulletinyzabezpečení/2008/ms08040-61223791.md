Security Bulletin

Microsoft Security Bulletin MS08-040 - Kritické
===============================================

Chyby zabezpečení serveru Microsoft SQL Server umožňují zvýšení úrovně oprávnění (941203)
-----------------------------------------------------------------------------------------

Publikováno: 8. července 2008 | Aktualizováno: 11. února 2009

**Verze:** 1.8

### Obecné informace

#### Shrnutí

Tato aktualizace zabezpečení řeší čtyři chyby zabezpečení oznámené soukromými osobami. Závažnější z těchto chyb zabezpečení mohou útočníkovi umožnit spuštění kódu a převzetí úplné kontroly nad postiženým systémem. Ověřený útočník by tak mohl instalovat programy, zobrazovat, měnit či odstraňovat data nebo vytvářet nové účty s úplnými právy správce.

Tato aktualizace zabezpečení je hodnocena jako důležitá pro podporované verze produktů SQL Server 7.0, SQL Server 2000, SQL Server 2005, Microsoft Data Engine (MSDE) 1.0, Microsoft SQL Server 2000 Desktop Engine (MSDE 2000), Microsoft SQL Server 2005 Express Edition, Microsoft SQL Server 2000 Desktop Engine (WMSDE) a Interní databáze systému Windows (WYukon). Další informace získáte v podčásti **Software obsahující tuto chybu a software neobsahující tuto chybu** tohoto oddílu.

Tato aktualizace zabezpečení řeší chyby zabezpečení změnou způsobu, jakým SQL Server spravuje opětovné použití stránky, přidělení více paměti pro funkci převodu, ověření souborů na disku před zavedením a ověření příkazů pro vkládání. Další informace o těchto chybách zabezpečení získáte v podčásti Nejčastější dotazy pro konkrétní položku chyby v části **Informace o chybách zabezpečení**.

**Doporučení:** Společnost Microsoft doporučuje, aby si zákazníci tuto aktualizaci aplikovali co možná nejdříve.

**Známé problémy.** [Článek 941203 znalostní báze Microsoft Knowledge Base](http://support.microsoft.com/kb/941203) popisuje aktuálně známé problémy, s nimiž se mohou zákazníci setkat v případě instalace této aktualizace zabezpečení. V článku jsou také uvedena doporučená řešení těchto problémů. Další informace o instalaci těchto aktualizací získáte v části **Nejčastější dotazy související s touto aktualizací zabezpečení** v této sekci.

#### Software obsahující tuto chybu a software neobsahující tuto chybu

Následující software byl testován za účelem zjištění, kterých verzí nebo edic se chyba týká. U dalších verzí nebo edic buď již skončila časově omezená podpora, nebo nejsou touto chybou postiženy. Pokud chcete zjistit dobu, po kterou je pro určitou verzi nebo edici softwaru poskytována podpora, navštivte web [Zásady poskytování technické podpory pro produkty společnosti Microsoft](http://go.microsoft.com/fwlink/?linkid=21742).

V závislosti na verzi nebo edici softwaru si můžete vybrat mezi níže uvedenými odkazy na aktualizace softwaru GDR a QFE a ručně nainstalovat aktualizaci z webu služby Stažení softwaru. Další informace o určení, kterou aktualizaci do systému nainstalujete, získáte v části **Nejčastější dotazy související s touto aktualizací zabezpečení** v této sekci.

Nejčastější dotazy související s touto aktualizací zabezpečení  
--------------------------------------------------------------
  

**Proč byl tento bulletin zabezpečení 12. srpna 2008 revidován?**  
Společnost Microsoft revidovala tento bulletin s cílem informovat o změně instalačního kódu aktualizace zabezpečení serveru SQL Server 2005 Service Pack 2, který řeší problém s instalací. Jedná se pouze o změnu instalačního kódu. Nebyly provedeny žádné změny binárních souborů aktualizace zabezpečení. I když se pro správce služby WSUS bude jednat o novou revizi této aktualizace, zákazníci, kteří již tuto aktualizaci zabezpečení úspěšně nainstalovali, nebudou muset aktualizaci znovu instalovat.
  
**Proč byl tento bulletin zabezpečení 23. července 2008 revidován?**  
Tento bulletin byl revidován přidáním informací o odinstalaci týkajících se modulu Microsoft SQL Server 2000 Desktop Engine (WMSDE) a interní databáze systému Windows (WYukon) upozorňujících na fakt, že odebrání aktualizace zabezpečení pro modul WMSDE či databázi WYukon zároveň způsobí úplnou odinstalaci těchto produktů ze systému.
  
**Proč byl tento bulletin 18. července 2008 revidován?**  
Tento bulletin byl revidován přidáním informace o změně rozpoznávání ve způsobu, jakým služba Windows Server Update Services (WSUS) nabízí aktualizaci pro modul Microsoft SQL Server 2000 Desktop Engine (WMSDE). Jedná se pouze o změnu rozpoznávání. Nebyly provedeny žádné změny binárních souborů.
  
**Proč byl tento bulletin 11. července 2008 revidován?**  
Tento bulletin byl revidován oznámením, že část **Známé problémy s touto aktualizací zabezpečení** v souvisejícím [článku 948110 znalostní báze Microsoft Knowledge Base](http://support.microsoft.com/kb/948110) byla aktualizována popisem problému se zjišťováním u upgradů identifikátoru SKU pro modul SQL Server 2000 Desktop Engine (WMSDE).
  
**Pro moji verzi serveru SQL se nabízí aktualizace softwaru GDR i QFE. Jak poznám, kterou aktualizaci použít?**  
Nejprve zjistěte číslo verze serveru SQL Server. Další informace o určení čísla verze serveru SQL Server získáte v [článku 321185 znalostní báze Microsoft Knowledge Base](http://support.microsoft.com/kb/321185).

**Proč není aktualizace softwaru QFE KB948108 poskytována pro server SQL Server 2005 Service Pack 2?**    
Jak je uvedeno [v článku 935897 znalostní báze Microsoft Knowledge Base](http://support.microsoft.com/kb/935897), opravy hotfix pro server SQL Server 2005 jsou kumulativní. Obvykle mají opravy hotfix kratší testovací období než soubory GDR (General Distribution Releases). Těm zákazníkům, kteří si nainstalovali buď dubnový [balíček kumulativních aktualizací 7 pro SQL Server 2005 Service Pack 2](http://support.microsoft.com/kb/949095), nebo červnový [balíček kumulativních aktualizací 8 pro SQL Server 2005 Service Pack 2](http://support.microsoft.com/kb/951217), nebude aktualizace softwaru QFE KB948108 nabídnuta, protože aktualizace zabezpečení dostupné prostřednictvím aktualizace softwaru QFE KB948108 již byly součástí dubnových i červnových kumulativních aktualizací. Tato aktualizace se však stále týká všech zákazníků, kteří si nenainstalovali ani dubnovou ani červnovou kumulativní aktualizaci pro SQL Server 2005 Service Pack 2.

**Referenční tabulky pro produkty SQL Server 2000 a MSDE 2000v části Informace o aktualizaci zabezpečení uvádějí, že některé aktualizace mohou vyžadovat restartování. Jak poznám, že zrovna moje aktualizace vyžaduje restartování?**    
Produkty SQL Server 2000 a MSDE 2000 mohou obsahovat závislé služby. Ty se po zastavení a restartování služby, na níž jsou závislé, nerestartují. Chyba zabezpečení týkající se opětovného použití paměťové stránky - CVE-2008-0085
---------------------------------------------------------------------------------


Byla zjištěna chyba zabezpečení ve způsobu, jakým SQL Server spravuje opětovné použití paměťové stránky. Útočník s přístupem databázového operátora, který tuto chybu zabezpečení úspěšně zneužije, může získat přístup k datům zákazníků.

Tuto chybu zabezpečení lze zobrazit jako standardní položku v seznamu známých chyb zabezpečení a ohrožení pod číslem [CVE-2008-0085](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-0085).

#### Skutečnosti snižující rizika chyby zabezpečení týkající se přístupu k informacím při opětovném použití paměťové stránky - CVE-2008-0085

Snížení rizika se týká nastavení, běžné konfigurace nebo obecného osvědčeného postupu existujícího ve výchozím stavu, který by mohl snížit závažnost zneužití chyby zabezpečení. V dané situaci mohou být užitečné následující faktory snížení rizika:

-   Tato chyba vyžaduje, aby měl útočník přístup databázového operátora k databázi, protokolu, záložním souborům databáze nebo záložním souborům protokolu.

#### Možná zástupná řešení chyby zabezpečení týkající se přístupu k informacím při opětovném použití paměťové stránky - CVE-2008-0085

Alternativní řešení se týká nastavení nebo změny konfigurace, která nevyřeší základní problém, ale pomůže před použitím aktualizace zablokovat známé útoky. Společnost Microsoft testovala následující alternativní řešení a v popisu uvádí, zda dané řešení omezuje funkčnost:

-   **Povolte možnost Common Criteria Compliance (Shoda se společnými kritérii) na serveru SQL 2005 Service Pack 2**

    Chcete-li povolit možnost [Common Criteria Compliance](http://msdn.microsoft.com/en-us/library/bb326650.aspx) (Shoda se společnými kritérii), proveďte při připojení k serveru SQL Server jako správce následující skript. Po spuštění skriptu restartujte SQL Server.

    ```
    sp_configure 'show advanced options', 1;
    GO
    RECONFIGURE;
    GO
    sp_configure 'common criteria compliance enabled', 1;
    GO
    RECONFIGURE
    GO
    ```

    **Dopad tohoto zástupného řešení:** Povolení možnosti Common Criteria Compliance (Shoda se společnými kritérii) má za následek mírné snížení výkonu.

    **Zrušení tohoto zástupného řešení:** Chcete-li možnost Common Criteria Compliance (Shoda se společnými kritérii) zakázat, proveďte při připojení k serveru SQL Server jako správce následující skript. Po spuštění skriptu restartujte SQL Server.

    ```
    sp_configure 'show advanced options', 1;
    GO
    RECONFIGURE;
    GO
    sp_configure 'common criteria compliance enabled', 0;
    GO
    RECONFIGURE
    GO
    ```

#### Nejčastější dotazy týkající se chyby zabezpečení spojené s přístupem k informacím při opětovném použití paměťové stránky - CVE-2008-0085

**Jaký je rozsah této chyby zabezpečení?**   
Byla zjištěna chyba zabezpečení ve způsobu, jakým SQL Server spravuje opětovné použití paměťové stránky. Útočník s přístupem databázového operátora, který by tuto chybu zabezpečení úspěšně zneužil, by mohl přistupovat k datům zákazníka, jejichž zobrazení by pro něj jinak bylo zakázané.

**Co tuto chybu zabezpečení způsobuje?**   
Tato chyba zabezpečení existuje ve způsobu, jakým SQL Server spravuje opětovné použití paměťové stránky. Při opětovném přidělování paměti se serveru SQL Server nepodaří inicializovat paměťové stránky.

**K čemu by mohl útočník tuto chybu zabezpečení zneužít?**   
Útočník by si mohl přečíst obsah paměťových stránek, které omezily data z relace jiného uživatele.

**Jak by mohl útočník tuto chybu zabezpečení zneužít?**   
Útočník s přístupem správce databáze by mohl sestavit neinicializované paměťové stránky z relace jiného uživatele nasměrováním zálohy do umístění, které tento útočník ovládá.

**Které systémy jsou touto chybou ohroženy nejvíce?**   
Nejvíce jsou ohroženy systémy, kde SQL Server používají nedůvěryhodné strany.

**Co je podstatou této aktualizace?**   
Tato aktualizace odstraňuje tuto chybu zabezpečení změnou způsobu, jakým SQL Server spravuje opětovné použití stránky.

**Jak mám odstranit zbývající data ze svých databází?**   
K dispozici je následující nová procedura uložená v systému, která slouží k fyzickému odstranění zbývajících dat z volné oblasti databázových stránek v prostředích, kde je ohroženo fyzické zabezpečení dat nebo záložní soubory:

```
sp_clean_db_free_space
```

Tato uložená procedura odebere zbývající informace ponechané na databázových stránkách v důsledku rutin změny dat na serveru SQL Server. Tato procedura smaže všechny stránky ve všech souborech v databázi.

```
sp_clean_db_free_space
@dbname = 'database_name',
[ @cleaning_delay = 'delay_in_seconds' ]
```

**Argumenty**

```
@dbname =  'database_name',
```

-   Název databáze ke smazání *dbname* je **nvarchar(384)** a nemůže mít hodnotu NULL.

```
[ @cleaning_delay = 'delay_in_seconds' ]
```

-   Aby se snížil dopad na vstupně-výstupní systém, lze mezi mazáním stránek určit zpoždění. Nastavení *cleaning\_delay* je **int** s výchozí hodnotou 0.

Následující příklad uvádí smazání všech zbývajících informací z databáze AdventureWorks.

```
USE Master
GO
EXEC sp_clean_db_free_space
@dbname = 'AdventureWorks';
```

Přetečení vyrovnávací paměti pro převod - CVE-2008-0086
-------------------------------------------------------

Byla zjištěna chyba zabezpečení ve funkci převodu na serveru SQL Server, která by ověřenému útočníkovi mohla umožnit získání zvýšení oprávnění. Útočník, který by tuto chybu zabezpečení úspěšně zneužil, by mohl spustit kód a převzít úplnou kontrolu nad systémem.

Tuto chybu zabezpečení lze zobrazit jako standardní položku v seznamu známých chyb zabezpečení a ohrožení pod číslem [CVE-2008-0086](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-0086).

#### Skutečnosti snižující závažnost rizika chyby zabezpečení týkající se přetečení vyrovnávací paměti pro převod - CVE-2008-0086

Snížení rizika se týká nastavení, běžné konfigurace nebo obecného osvědčeného postupu existujícího ve výchozím stavu, který by mohl snížit závažnost zneužití chyby zabezpečení. V dané situaci mohou být užitečné následující faktory snížení rizika:
