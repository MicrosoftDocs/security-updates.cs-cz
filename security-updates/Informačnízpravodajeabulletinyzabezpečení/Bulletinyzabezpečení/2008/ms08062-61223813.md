Security Bulletin

Microsoft Security Bulletin MS08-062 - Kritické
===============================================

Chyba zabezpečení ve službě Tisk přes Internet systému Windows umožňuje vzdálené spuštění kódu (953155)
-------------------------------------------------------------------------------------------------------

Publikováno: 14. října 2008 | Aktualizováno: 29. října 2008

**Verze:** 2.2

### Obecné informace

#### Shrnutí

Tato aktualizace řeší soukromě oznámenou chybu zabezpečení služby Tisk přes Internet systému Windows, která může způsobit vzdálené spuštění kódu. Útočník, který by tuto chybu zabezpečení zneužil, by mohl získat úplnou kontrolu nad postiženým systémem. Útočník by tak mohl instalovat programy, zobrazovat, měnit či odstraňovat data nebo vytvářet nové účty.

Tato aktualizace je označena jako důležitá pro všechny podporované verze systému Microsoft Windows 2000, Windows XP, Windows Server 2003 a Windows Server 2008. Další informace získáte v podčásti **Software obsahující tuto chybu a software neobsahující tuto chybu** tohoto oddílu.

Tato aktualizace zabezpečení řeší uvedenou chybu zabezpečení změnou způsobu přidělování paměti v rámci služby IPP (Internet Printing Protocol). Další informace o této chybě zabezpečení získáte v podčásti Nejčastější dotazy pro konkrétní položku chyby v části **Informace o chybách zabezpečení**.

**Doporučení:** Společnost Microsoft doporučuje, aby si zákazníci tuto aktualizaci aplikovali co možná nejdříve.

**Známé problémy.** Žádné

**Proč byl tento bulletin 16. října 2008 revidován?**
Společnost Microsoft revidovala tento bulletin s cílem upřesnit, že služba Tisk přes Internet systému Windows je spuštěna v kontextu zařazovací služby, která je spuštěna se systémovými oprávněními. Útočník, který by chybu zabezpečení popsanou v tomto bulletinu úspěšně zneužil, by mohl spustit libovolný kód s vyššími oprávněními a získat tak úplnou kontrolu nad postiženým systémem.

Protože však systém Windows Server 2008 pro systémy s procesorem Itanium nebo systém Windows Vista nepodporuje službu Tisk přes Internet, není vystaven útokům uvedeným v tomto bulletinu zabezpečení. V rámci zvýšení ochrany proti eventuálním další útokům by měli všichni uživatelé systému Windows Server 2008 pro systémy s procesorem Itanium a systému Windows Vista tuto aktualizaci zabezpečení použít.

Chyba zabezpečení způsobující přetečení celých čísel ve službě IPP - CVE-2008-1446  
----------------------------------------------------------------------------------
  
<span></span>  
V systémech Windows, ve kterých je spuštěna služba IIS (Internet Information Services) s povolenou službou tisku přes Internet, byla zjištěna chyba zabezpečení umožňující vzdálené spuštění kódu. Uvedená chyba zabezpečení umožňuje vzdálenému ověřenému útočníkovi spustit v postiženém systému libovolný kód.
  
Tuto chybu zabezpečení lze zobrazit jako standardní položku v seznamu známých chyb zabezpečení a ohrožení pod číslem [CVE-2008-1446](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-1446).
  
#### Skutečnosti snižující závažnost rizika spojeného s chybou zabezpečení způsobující přetečení celých čísel ve službě IPP - CVE-2008-1446
  
Snížení rizika se týká nastavení, běžné konfigurace nebo obecného osvědčeného postupu existujícího ve výchozím stavu, který by mohl snížit závažnost zneužití chyby zabezpečení. V dané situaci mohou být užitečné následující faktory snížení rizika:
  
-   Protokol IPP (Internet Printing Protocol) je ve výchozím nastavení povolen u služby IIS v systému Microsoft Windows 2000 a Windows XP. Výchozí oprávnění souborů jsou však nastavena tak, že přístup k protokolu IPP je povolen pouze uživatelům skupiny Authenticated. Vzhledem k tomuto omezení nejsou neověřeným uživatelům ve výchozím nastavení udělena nezbytná oprávnění pro složky, takže nemohou tuto chybu zabezpečení vzdáleně zneužít.  
-   Instalace služby IIS v systému Windows Server 2003 nejsou ve výchozím nastavení uvedenou chybou ohroženy. Ohroženy jsou pouze instalace systému Windows Server 2003, ve kterých je v rámci služby IIS povolen filtr ISAPI protokolu IPP. Součásti Tisk přes Internet i Stránky ASP jsou nezbytné k webové správě tiskárny a tisku prostřednictvím Internetu. Pokud je součást Stránky ASP nastavena na možnost Povoleno a součást Tisk přes Internet na možnost Zakázáno, je povolena pouze webová správa tiskárny. Uživatelé se nemohou prostřednictvím služby Tisk přes Internet připojit k žádné sdílené tiskárně.  
-   Instalace služby IIS v systému Windows Server 2008 nejsou ve výchozím nastavení ohroženy. Počítač se systémem Windows Server 2008 může fungovat jako klientský počítač využívající funkci tisku přes Internet. Tiskový server musí ověřit uživatele počítače, kteří vytvořili požadavky na tisk, ještě před tím, než mohou použít jakoukoli tiskárnu připojenou k danému serveru. Chcete-li nastavit počítač se systémem Windows Server 2008 jako server podporující tisk přes Internet, je nutné nainstalovat roli Webový server (IIS), roli Tiskové služby a roli služby Tisk přes Internet v rámci role Tiskové služby.  
-   Zneužití dané chyby zabezpečení je možné pouze v případě, že server IIS používá při odchozí komunikaci se škodlivým počítačem port 445. Správci, kteří blokují odchozí připojení SMB ze serveru IIS do Internetu, mohou nedůvěryhodným systémům zabránit ve zneužití uvedené chyby zabezpečení.
  
#### Možná zástupná řešení chyby zabezpečení způsobující přetečení celých čísel ve službě IPP - CVE-2008-1446
  
Alternativní řešení se týká nastavení nebo změny konfigurace, která nevyřeší základní problém, ale pomůže před použitím aktualizace zablokovat známé útoky. Společnost Microsoft testovala následující alternativní řešení a v popisu uvádí, zda dané řešení omezuje funkčnost:
  
-   **Zakázání služby IPP**
  
    Zakázání služby IPP ve službě IIS 5.0 a IIS 5.1
  
    1.  Spusťte **Správce služeb sítě Internet**.  
    2.  Klikněte pravým tlačítkem myši na **výchozí web** a vyberte položku **Vlastnosti**.  
    3.  Přejděte na kartu **Domovský adresář** a vyberte možnost **Konfigurace**.  
    4.  Projděte seznam až dolů a vyhledejte příponu **.printer** spolu s odpovídající cestou ke knihovně **msw3prt.dll**.  
    5.  Klikněte na tlačítko **Odebrat**.  
    6.  Klikněte na tlačítko **OK**.  
    7.  Restartujte službu IIS.
  
    **Poznámka:** Službu Tisk přes Internet lze nakonfigurovat jak pomocí zásad skupiny, tak pomocí Správce služeb sítě Internet. Pokud dojde ke konfliktu mezi nastavením zásad skupiny a nastavením ve Správci služeb sítě Internet, nastavení zásad skupiny mají vždy přednost. Pokud ve Správci služeb sítě Internet odstraníte možnost Tisk přes Internet, zkontrolujte, že nedojde k jejímu opětovnému povolení prostřednictvím místních zásad skupiny nebo zásad skupin domény. (Výchozí nastavení zásad skupiny možnost Tisk přes Internet nepovoluje ani nezakazuje.) V modulu Zásad skupiny MMC snap-in zkontrolujte Nastavení systému Windows | Šablony pro správu | Tisk | Tisk prostřednictvím Internetu.
  
    Zakázání služby IPP ve službě IIS 6.0
  
    1.  Spusťte **Správce služby IIS** nebo modul **snap-in služby IIS**.  
    2.  Rozbalte položku **\* název\_serveru**, kde hodnota název\_serveru představuje název daného serveru.

#### Nejčastější dotazy týkající se chyby zabezpečení způsobující přetečení celých čísel ve službě IPP - CVE-2008-1446
  
**Jaký je rozsah této chyby zabezpečení?**  
Jedná se o chybu zabezpečení umožňující vzdálené spuštění kódu. Útočník, který by tuto chybu zabezpečení zneužil, by mohl získat úplnou kontrolu nad postiženým systémem. Útočník by tak mohl instalovat programy, zobrazovat, měnit či odstraňovat data nebo vytvářet nové účty.
  
**Co tuto chybu zabezpečení způsobuje?**  
Funkce protokolu IPP (Internet Printing Protocol) je implementována jako rozšíření ISAPI pro službu IIS. V implementaci protokolu IPP na serverech se systémem Windows a službou IIS byla zjištěna chyba zabezpečení zneužitelná ověřeným uživatelem prostřednictvím vzdáleného přístupu. V implementaci protokolu IPP bylo zjištěno přetečení celých čísel, které by mohlo ověřenému útočníkovi umožnit vzdálené spuštění kódu u ohrožených serverů IIS.
  
**Co je rozhraní ISAPI?**  
Rozhraní ISAPI (Internet Services Application Programming Interface) je technologie umožňující webovým vývojářům rozšířit funkce webových serverů tím, že vytvoří vlastní kód, který zajišťuje nové služby pro webový server. Vlastní kód může být implementován jako filtr ISAPI (nová funkce zajišťuje službu nižší úrovně) nebo jako rozšíření ISAPI (nová funkce zajišťuje službu vyšší úrovně). V tomto případě představuje ohrožený kód rozšíření ISAPI.
  
**Na které rozšíření ISAPI se daná chyba zabezpečení vztahuje?**  
Ohroženo je rozšíření ISAPI, které implementuje protokol IPP (Internet Printing Protocol, standardní protokol definovaný v dokumentech RFC 2910 a 2911). Protokol IPP umožňuje vytvářet požadavky na tiskové služby a zjišťovat stav tiskových úloh přes Internet prostřednictvím protokolu HTTP. Uživatel na služební cestě může například pomocí protokolu IPP odeslat přes Internet tiskovou úlohu, kterou chce vytisknout na tiskárně v podnikové síti. Může také zjistit, zda byl požadavek na tisk proveden bez chyby.
  
Systém Microsoft Windows 2000 zavedl nativní podporu služby Tisk přes Internet. Implementace systému Microsoft Windows 2000 uživatelům umožňuje přímý tisk na adresu URL a zobrazení informací o tiskových úlohách prostřednictvím prohlížečů. Podpora služby Tisk přes Internet je ve verzích systému Microsoft Windows 2000 Server povolena ve výchozím nastavení. Lze ji také nainstalovat jako volitelnou součást systému Microsoft Windows 2000 Professional, Windows XP, Windows Server 2003 a Windows Server 2008.
  
**K čemu by mohl útočník tuto chybu zabezpečení zneužít?**  
Útočník, který by tuto chybu zabezpečení úspěšně zneužil, by mohl v systému uživatele spustit libovolný kód se systémovými oprávněními. Útočník, který by tuto chybu zabezpečení zneužil, by mohl získat úplnou kontrolu nad postiženým systémem. Útočník by tak mohl instalovat programy, zobrazovat, měnit či odstraňovat data nebo vytvářet nové účty.
  
**Jak by mohl útočník tuto chybu zabezpečení zneužít?**  
Útočník by mohl na ohrožený webový server odeslat speciálně vytvořený požadavek HTTP POST zajišťující připojení serveru k počítači, který ovládá útočník a který funguje jako tiskárna pomocí protokolu IPP. Útočník by pak prostřednictvím speciálně vytvořených odpovědí spustil na cílovém serveru vzdálený kód.
  
**Které systémy jsou touto chybou ohroženy nejvíce?**  
Nejvíce jsou touto chybou zabezpečení ohroženy pracovní stanice a servery používající službu IIS s povolenou službou IPP (Internet Printing Protocol).
  
**Co je podstatou této aktualizace?**  
Tato aktualizace zabezpečení řeší uvedenou chybu zabezpečení ověřováním dat přijatých ze vzdálených počítačů v rámci služby IPP (Internet Printing Protocol).

Ano, společnost Microsoft získala informace o cílených útocích, které se snaží tuto chybu zabezpečení zneužít. V době původního zveřejnění tohoto bulletinu zabezpečení však nezaznamenala žádné důkazy nebo publikované příklady kódu.

**Ověření instalace aktualizace**

-   **Nástroj Microsoft Baseline Security Analyzer**

    Pokud chcete ověřit, je-li tato aktualizace zabezpečení v počítači nainstalována, můžete použít nástroj Microsoft Baseline Security Analyzer (MBSA). Další informace naleznete v předchozí části tohoto bulletinu **Nástroje a doporučené postupy zjišťování a nasazení**.

-   **Ověření verze souboru**

    Vzhledem k různým vydáním systému Microsoft Windows se následující postup může lišit od postupu ve vašem systému. Pokud tomu tak je, bude třeba před dokončením následujících kroků konzultovat dokumentaci k produktu.

    1.  Klikněte na tlačítko **Start** a potom na příkaz **Hledat**.
    2.  Ve skupinovém rámečku **Průvodce vyhledáváním** v podokně **Výsledky hledání** klikněte na odkaz **Všechny soubory a složky**.
    3.  Do pole **Část nebo celý název souboru** zadejte název souboru z příslušné tabulky a klikněte na tlačítko **Hledat**.
    4.  V seznamu souborů klikněte pravým tlačítkem myši na název souboru z příslušné tabulky s informacemi o souborech a klikněte na příkaz **Vlastnosti**.
        **Poznámka** V závislosti na vydání operačního systému nebo nainstalovaných programech nemusejí být některé soubory uvedené v tabulce s informacemi o souborech nainstalovány.
    5.  Na kartě **Verze** zjistěte verzi souboru nainstalovaného v systému a porovnejte ji s verzí popsanou v příslušné tabulce s informacemi o souborech.
        **Poznámka:** Atributy (kromě verze souboru) se mohou při instalaci změnit. Porovnání jiných atributů souboru s informacemi v příslušné tabulce není spolehlivou metodou ověření instalace aktualizace. V určitých případech mohou být soubory při instalaci přejmenovány. Pokud soubor nebo informace o verzi nejsou k dispozici, použijte k ověření instalace aktualizace některou jinou metodu.

-   **Ověření klíče registru**

    Úspěšnou instalaci této aktualizace zabezpečení lze také ověřit revizí klíčů registru uvedených v **referenční tabulce** v této části.

    Tyto klíče registru nemusí obsahovat úplný seznam nainstalovaných souborů. Tyto klíče registru nemusí být vytvořeny správně, pokud správce nebo výrobce OEM integruje nebo vloží aktualizaci zabezpečení do zdrojových instalačních souborů systému Windows.
