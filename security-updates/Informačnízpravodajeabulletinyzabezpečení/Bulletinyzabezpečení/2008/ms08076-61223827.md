Microsoft Security Bulletin MS08-076 - Kritické
===============================================

Chyby zabezpečení součástí služby Windows Media umožňují vzdálené spuštění kódu (959807)
----------------------------------------------------------------------------------------

Publikováno: 9. prosince 2008 | Aktualizováno: 25. února 2009

**Verze:** 3.1

### Obecné informace

#### Shrnutí

Tato aktualizace zabezpečení řeší dvě chyby zabezpečení oznámené soukromými osobami, a to v následujících součástech služby Windows Media: Windows Media Player, Windows Media Format Runtime a Windows Media Services. Nejzávažnější chyba zabezpečení by mohla umožnit vzdálené spuštění kódu. Pokud by byl uživatel přihlášen s uživatelskými právy správce, mohl by útočník, který by tuto chybu zabezpečení zneužil, získat úplnou kontrolu nad postiženým systémem. Útočník by tak mohl instalovat programy, zobrazovat, měnit či odstraňovat data nebo vytvářet nové účty s úplnými uživatelskými právy. Uživatelé, jejichž účty jsou konfigurovány tak, aby měli v systému méně uživatelských práv, by byli vystaveni menšímu riziku než uživatelé s uživatelskými právy správce.

Tato aktualizace zabezpečení je hodnocena jako důležitá pro program Windows Media Player 6.4, součásti Windows Media Format Runtime 7.1, Windows Media Format Runtime 9.0, Windows Media Format Runtime 9.5, Windows Media Format Runtime 11, službu Windows Media Services 4.1, Windows Media Services 9 Series a Windows Media Services 2008. Další informace získáte v podčásti tohoto oddílu nazvané **Software obsahující tuto chybu a software neobsahující tuto chybu**.

Tato aktualizace zabezpečení řeší první chybu zabezpečení změnou způsobu, jakým služba Windows Media ověřuje platnost odpovědí na ověřovací dotazy. Tato aktualizace zabezpečení řeší druhou chybu zabezpečení zajištěním toho, že klienti služby Windows Media zacházejí se servery používajícími adresy ISATAP jako s externími. Další informace o těchto chybách zabezpečení získáte v podčásti Nejčastější dotazy pro konkrétní položku chyby v části **Informace o chybách zabezpečení**.

**Doporučení:** Společnost Microsoft doporučuje, aby si zákazníci tuto aktualizaci aplikovali co možná nejdříve.

**Známé problémy.** Žádné

#### Software obsahující tuto chybu a software neobsahující tuto chybu

Následující software byl testován za účelem zjištění, kterých verzí nebo edic se chyba týká. U dalších verzí nebo edic buď již skončila časově omezená podpora, nebo nejsou touto chybou postiženy. Pokud chcete zjistit dobu, po kterou je pro určitou verzi nebo edici softwaru poskytována podpora, navštivte web [Zásady poskytování technické podpory pro produkty společnosti Microsoft](http://go.microsoft.com/fwlink/?linkid=21742).

Nejčastější dotazy související s touto aktualizací zabezpečení
--------------------------------------------------------------


**Proč byla 13. ledna 2009 vydána přepracovaná verze tohoto bulletinu?**  
Společnost Microsoft přepracovala tento bulletin, aby nabídla nové balíčky aktualizací pro součást Windows Media Format Runtime 9.5 v systémech Windows XP Service Pack 2 (KB952069) a Windows XP Service Pack 3 (KB952069). Nové balíčky aktualizací opravují problém s instalací, jehož následkem byla nedokončená instalace v systému Windows XP se součástí Windows Media Format Runtime 9.5, pokud zákazník nejprve nainstaloval samostatnou aktualizaci KB944110 před instalací aktualizace KB952069. Takoví zákazníci jsou i nadále vystaveni ohrožení a měli by si stáhnout a nainstalovat přepracované balíčky aktualizací. Toto přepracování se netýká žádných dalších balíčků aktualizací. Zákazníci, kteří provozují všechny ostatní podporované verze programu Windows Media Player, součásti Windows Media Format Runtime a služby Windows Media Services s touto chybou a úspěšně nainstalovali původní balíček aktualizace zabezpečení, nemusí provádět žádnou další akci. Přepracované balíčky aktualizace zabezpečení lze získat prostřednictvím stejných distribučních kanálů jako původní aktualizace zabezpečení, včetně funkce Automatické aktualizace, webu Windows Update a služby Windows Server Update Services.

Společnost Microsoft také revidovala tento bulletin a uvedla program Windows Media Player 6.4 a službu Windows Media Services 4.1 mezi postiženými ve všech edicích systému Microsoft Windows 2000 Service Pack 4 (nejedná se pouze o systém Microsoft Windows 2000 Server Service Pack 4). Lze použít stejnou aktualizaci. Pro tento operační systém nebyl vytvořen žádný nový balíček aktualizací. Zákazníci, kterým byla nabídnuta aktualizace KB954600 pro program Windows Media Player 6.4 nebo KB952068 pro službu Windows Media Services 4.1 a nenainstalovali ji, by tak však měli učinit.

**Kde jsou podrobné informace o souborech?**  
Podrobné informace o souborech je možné získat v [článku 959807 znalostní báze Microsoft Knowledge Base](http://support.microsoft.com/kb/959807).

**Proč se tato aktualizace zabývá několika zjištěnými chybami zabezpečení?**  
Tato aktualizace obsahuje podporu spojenou s několika chybami zabezpečení, protože změny nutné k odstranění daných problémů je třeba provést v souvisejících souborech. Zákazníci nemusí instalovat několik téměř shodných aktualizací, ale pouze tuto aktualizaci.

**Používám starší vydání softwaru uvedeného v tomto bulletinu zabezpečení. Jak mám postupovat?**  
Software uvedený v tomto bulletinu byl testován za účelem zjištění, kterých verzí se chyba týká. U ostatních vydání již skončila časově omezená podpora. Pokud chcete zjistit dobu, po kterou je pro určité vydání softwaru poskytována podpora, navštivte web [Zásady poskytování technické podpory pro produkty společnosti Microsoft](http://go.microsoft.com/fwlink/?linkid=21742).

Zákazníkům používajícím starší vydání softwaru doporučujeme přejít na podporovaná vydání, aby jejich počítače byly před možnými chybami zabezpečení lépe chráněny. Chyba zabezpečení hlavního názvu služby (SPN) - CVE-2008-3009
-------------------------------------------------------------


V součástech služby Windows Media je chyba zabezpečení reflexe pověření pro přihlášení, která by mohla útočníkovi umožnit spuštění kódu se stejnými právy jako má místní uživatel nebo s pověřením distribuce služby Windows Media Services. Tato chyba zabezpečení existuje kvůli slabině v implementaci hlavního názvu služby (SPN) pro součásti služby Windows Media.

Tuto chybu zabezpečení lze zobrazit jako standardní položku v seznamu známých chyb zabezpečení a ohrožení pod číslem [CVE-2008-3009](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-3009).

#### Skutečnosti snižující závažnost rizika spojeného s chybou zabezpečení hlavního názvu služby (SPN) - CVE-2008-3009

Snížení rizika se týká nastavení, běžné konfigurace nebo obecného osvědčeného postupu existujícího ve výchozím stavu, který by mohl snížit závažnost zneužití chyby zabezpečení. V dané situaci mohou být užitečné následující faktory snížení rizika:

-   Služba Windows Media Services je volitelná součást pro všechny platformy Windows Server a není nainstalována ve výchozím nastavení.
-   V případě útoku z webu by útočník musel být hostitelem webu s webovou stránkou, pomocí které lze tuto chybu zabezpečení zneužít. Útočník nemůže žádným způsobem přinutit uživatele k návštěvě škodlivé webové stránky. Musel by je přesvědčit, aby navštívili konkrétní web. K tomu se obvykle používá odkaz, na který uživatel klikne a přejde tak na útočníkův web. Po kliknutí na takový odkaz by byl uživatel vyzván k provedení několika akcí. K útoku by mohlo dojít až po provedení těchto akcí.
-   V systémech Windows Server 2008 a Windows Server 2008 pro systémy s procesorem x64 je součást Windows Media Format Runtime 11 dostupná pouze v těch systémech, které byly nainstalovány s funkcí Možnosti práce s počítačem.

#### Možná zástupná řešení chyby zabezpečení hlavního názvu služby (SPN) - CVE-2008-3009

Společnost Microsoft zatím nenalezla žádné zástupné řešení této chyby zabezpečení.

#### Nejčastější dotazy týkající se chyby zabezpečení hlavního názvu služby (SPN) - CVE-2008-3009

**Jaký je rozsah této chyby zabezpečení?**  
Jedná se o chybu zabezpečení umožňující vzdálené spuštění kódu. Útočník, který by úspěšně zneužil tuto chybu zabezpečení, by mohl spustit kód se stejnými právy jako přihlášený uživatel. Pokud by byl uživatel přihlášen s uživatelskými právy správce, mohl by útočník, který by tuto chybu zabezpečení zneužil, získat úplnou kontrolu nad postiženým systémem. Útočník by tak mohl instalovat programy, zobrazovat, měnit či odstraňovat data nebo vytvářet nové účty s úplnými uživatelskými právy. Uživatelé, jejichž účty jsou konfigurovány tak, aby měli v systému méně uživatelských práv, by byli vystaveni menšímu riziku než uživatelé s uživatelskými právy správce. Útočník, který úspěšně zneužije tuto chybu zabezpečení, by také mohl spustit kód s pověřením distribuce služby Windows Media Services. V tomto případě by mohl útočník spustit kód se stejnými právy, jaká má účet SÍŤOVÉ SLUŽBY.

**Co tuto chybu zabezpečení způsobuje?**  
Součásti služby Windows Media neprovedou výslovný souhlas s ochranou proti reflexi pověřovacích údajů NTLM, jejímž cílem je zajistit, aby pověření uživatele nebyla přehrána zpět na systém uživatele a použita ke zosobnění uživatele.

**Co je hlavní název služby (SPN)?**  
Hlavní název služby (SPN) je název, kterým klient jedinečně identifikuje instanci služby. Název SPN poskytuje typ identity služby používaný při ověřování služby pro klienta. Když se chce klient připojit ke službě, lokalizuje se instance služby, složí se název SPN pro instanci, připojí se ke službě a uvede název SPN za účelem ověření služby. Další informace o hlavním názvu služby (SPN) naleznete v článku [Identita a ověřování služby](http://msdn.microsoft.com/en-us/library/ms733130.aspx). Další informace o tom, jak by měl být formátován hlavní název služby (SPN), naleznete v článku [Formáty názvů pro jedinečné hlavní názvy služby (SPN)](http://msdn.microsoft.com/en-us/library/ms677601(vs.85).aspx).

**K čemu by mohl útočník tuto chybu zabezpečení zneužít?**  
Útočník, který by úspěšně zneužil tuto chybu zabezpečení, by mohl získat stejná uživatelská práva, jaká má přihlášený uživatel. Pokud by byl uživatel přihlášen s uživatelskými právy správce, mohl by útočník, který by tuto chybu zabezpečení zneužil, získat úplnou kontrolu nad postiženým systémem. Útočník by tak mohl instalovat programy, zobrazovat, měnit či odstraňovat data nebo vytvářet nové účty s úplnými uživatelskými právy. Uživatelé, jejichž účty jsou konfigurovány tak, aby měli v systému méně uživatelských práv, by byli vystaveni menšímu riziku než uživatelé s uživatelskými právy správce.

**Jak by mohl útočník tuto chybu zabezpečení zneužít?**  
Tato chyba zabezpečení vyžaduje, aby uživatel s postiženou verzí součásti služby Windows Media navštívil škodlivý server. Útočník by mohl být hostitelem škodlivého sdíleného serveru nebo webové stránky, která vyžaduje ověřování. Když se klient připojí ke škodlivému sdílenému serveru a provede ověřování, může útočník použít klientem poskytnuté přihlašovací údaje nebo předat přihlašovací údaje na další servery, a tím napodobit klienta. Útočník nemůže žádným způsobem přinutit uživatele k návštěvě speciálně vytvořené sdílené položky na síti nebo webové stránky. Místo toho by útočník musel přesvědčit uživatele, aby navštívili takovou speciálně vytvořenou sdílenou položku na síti nebo webovou stránku. K tomu se obvykle používá odkaz v e-mailu nebo zpráva programu pro zasílání rychlých zpráv, na které uživatelé kliknou a přejdou tak na útočníkův web.

**Které systémy jsou touto chybou ohroženy nejvíce?**  
Každý systém, který používá postiženou verzi programu Windows Media Player, součásti Windows Media Format Runtime nebo služby Windows Media Services je vystaven riziku spojenému s touto chybou zabezpečení. Touto chybou jsou nejvíce ohroženy systémy, v nichž se často používají tyto součásti, což jsou především pracovní stanice a terminálové servery.

**Co je podstatou této aktualizace?**  
Tato aktualizace zabezpečení řeší uvedenou chybu zabezpečení změnou způsobu, jakým součást služby Windows Media ověřuje platnost odpovědí na ověřovací dotazy a brání tak útokům pomocí přehraných pověření pro přihlášení.

**Byla v době zveřejnění tohoto bulletinu uvedená chyba zabezpečení veřejně známá?**  
Ne. Společnost Microsoft byla o této chybě informována důvěryhodným zdrojem. Společnost Microsoft nemá žádné informace, které by naznačovaly, že tato chyba zabezpečení byla v době vydání tohoto bulletinu veřejně známá.

**Měla v době zveřejnění tohoto bulletinu společnost Microsoft nějaké informace o tom, že tato chyba zabezpečení byla zneužita?**  
Ne. Společnost Microsoft neměla žádné informace, které by naznačovaly, že daná chyba zabezpečení byla v době původního zveřejnění tohoto bulletinu zneužita k útoku na zákazníky, a nezaznamenala ani žádné důkazy nebo publikované příklady kódu, které by naznačovaly, že tato chyba byla zneužita.

Chyba zabezpečení rozhraní ISATAP - CVE-2008-3010
-------------------------------------------------


V podporovaných verzích součástí služby Windows Media byla zjištěna chyba zabezpečení umožňující zpřístupnění informací, která by mohla vést k odhalení přihlašovacích údajů nástroje NTLM. Každá součást služby Windows Media, která přistoupí na adresu URL využívající adresu ISATAP, může propustit přihlašovací údaje uživatele nástroje NTLM na server hostitele dané adresy URL. To by mohlo útočníkovi, který je vně zóny intranet, umožnit shromažďování přihlašovacích údajů nástroje NTLM v podnikovém prostředí.

Tuto chybu zabezpečení lze zobrazit jako standardní položku v seznamu známých chyb zabezpečení a ohrožení pod číslem [CVE-2008-3010](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-3010).

#### Skutečnosti snižující závažnost rizika spojeného s chybou zabezpečení rozhraní ISATAP - CVE-2008-3010

Snížení rizika se týká nastavení, běžné konfigurace nebo obecného osvědčeného postupu existujícího ve výchozím stavu, který by mohl snížit závažnost zneužití chyby zabezpečení. V dané situaci mohou být užitečné následující faktory snížení rizika:

-   Doporučené postupy pro používání brány firewall a její standardní výchozí konfigurace zajišťují ochranu síťových prostředků před útoky pocházejícími z oblasti mimo rozlehlou síť. Je vhodné, aby systémy připojené k Internetu měly přístupný minimální počet portů a protokolů.
-   V případě útoku z webu by útočník musel být hostitelem webu s webovou stránkou, pomocí které lze tuto chybu zabezpečení zneužít. Útočník nemůže žádným způsobem přinutit uživatele k návštěvě škodlivé webové stránky. Musel by je přesvědčit, aby navštívili konkrétní web. K tomu se obvykle používá odkaz, na který uživatel klikne a přejde tak na útočníkův web. Po kliknutí na takový odkaz by byl uživatel vyzván k provedení několika akcí. K útoku by mohlo dojít až po provedení těchto akcí.
-   V systémech Windows Server 2008 a Windows Server 2008 pro systémy s procesorem x64 je součást Windows Media Format Runtime 11 dostupná pouze v těch systémech, které byly nainstalovány s funkcí Možnosti práce s počítačem.

#### Možná zástupná řešení chyby zabezpečení rozhraní ISATAP - CVE-2008-3010

Alternativní řešení se týká nastavení nebo změny konfigurace, která nevyřeší základní problém, ale pomůže před použitím aktualizace zablokovat známé útoky. Společnost Microsoft testovala následující alternativní řešení a v popisu uvádí, zda dané řešení omezuje funkčnost:

-   **Protokol blokování adres IP typ 41 (ISATAP) na bráně firewall**

    Protokol rozhraní ISATAP je definován jako protokol typu 41 podle části [RFC4214: Intra-Site Automatic Tunnel Addressing Protocol (ISATAP)](http://www.ietf.org/rfc/rfc4214.txt). Blokování tohoto typu protokolu bránou firewall umožní zabezpečení systémů za touto bránou před zneužitím této chyby zabezpečení. Společnost Microsoft doporučuje blokovat veškerou nevyžádanou příchozí komunikaci z Internetu, a zabránit tak útokům využívajícím jiné porty nebo protokoly.

#### Nejčastější dotazy týkající se chyby zabezpečení rozhraní ISATAP - CVE-2008-3010

**Jaký je rozsah této chyby zabezpečení?**  
Jedná se o chybu zabezpečení typu zpřístupnění informací. Útočník, který by úspěšně zneužil tuto chybu zabezpečení, by mohl získat přihlašovací údaje nástroje NTLM přihlášeného uživatele. Tyto přihlašovací údaje by následně mohly být přehrány zpět k systému uživatele, a tím spustit kód se stejnými právy, jaké má přihlášený uživatel.

**Co tuto chybu zabezpečení způsobuje?**  
Během připojování k serveru, který používá adresy ISATAP, nebudou postižené součásti služby Windows Media zacházet správně se zdroji jako s částí zóny Místní intranet.

**Co je ISATAP?**  
Služba tunelového připojení ISATAP (Intra-Site Automatic Tunnel Addressing Protocol) poskytuje připojení protokolem IPv6 v rámci sítě intranet protokolu IPv4. Standard RFC pro službu ISATAP je provozován pracovní skupinou pro internetový koncept organizace Internet Engineering Task Force (IETF). Další informace o službě ISATAP získáte v části [RFC4214: Intra-Site Automatic Tunnel Addressing Protocol (ISATAP)](http://www.ietf.org/rfc/rfc4214.txt).

**Co je NTLM?**  
NTLM je ověřovací protokol založený na mechanismu výzva/odpověď.

**K čemu by mohl útočník tuto chybu zabezpečení zneužít?**  
Útočník, který by tuto chybu úspěšně zneužil, by mohl získat uživatelovy přihlašovací údaje nástroje NTLM. Tyto přihlašovací údaje by následně mohly být přehrány zpět k systému uživatele, a tím spustit kód se stejnými právy, jaké má přihlášený uživatel.

**Jak by mohl útočník tuto chybu zabezpečení zneužít?**  
Tato chyba zabezpečení vyžaduje, aby uživatel s postiženou verzí součásti služby Windows Media navštívil škodlivý server. Útočník by mohl být hostitelem škodlivého sdíleného serveru nebo webové stránky, která vyžaduje ověřování. Když se klient připojí ke škodlivému sdílenému serveru a provede ověřování, může útočník použít klientem poskytnuté přihlašovací údaje nebo předat přihlašovací údaje na další servery, a tím napodobit klienta. Útočník nemůže žádným způsobem přinutit uživatele k návštěvě speciálně vytvořené sdílené položky na síti nebo webové stránky. Místo toho by útočník musel přesvědčit uživatele, aby navštívili takovou speciálně vytvořenou sdílenou položku na síti nebo webovou stránku. K tomu se obvykle používá odkaz v e-mailu nebo zpráva programu pro zasílání rychlých zpráv, na které uživatelé kliknou a přejdou tak na útočníkův web.

**Které systémy jsou touto chybou ohroženy nejvíce?**  
Každý systém, který používá postiženou verzi programu Windows Media Player, součásti Windows Media Format Runtime nebo služby Windows Media Services je vystaven riziku spojenému s touto chybou zabezpečení. Touto chybou jsou nejvíce ohroženy systémy, v nichž se často používají tyto součásti, což jsou především pracovní stanice a terminálové servery.

**Co je podstatou této aktualizace?**  
Tato aktualizace odstraní tuto chybu zabezpečení zajištěním toho, že klienti součásti služby Windows Media zacházejí se servery používajícími adresy ISATAP jako s externími a nepředávají přihlašovací údaje uživatele.

**Byla v době zveřejnění tohoto bulletinu uvedená chyba zabezpečení veřejně známá?**  
Ne. Společnost Microsoft byla o této chybě informována důvěryhodným zdrojem. Společnost Microsoft nemá žádné informace, které by naznačovaly, že tato chyba zabezpečení byla v době vydání tohoto bulletinu veřejně známá.

**Měla v době zveřejnění tohoto bulletinu společnost Microsoft nějaké informace o tom, že tato chyba zabezpečení byla zneužita?**  
Ne. Společnost Microsoft neměla žádné informace, které by naznačovaly, že daná chyba zabezpečení byla v době původního zveřejnění tohoto bulletinu zneužita k útoku na zákazníky, a nezaznamenala ani žádné důkazy nebo publikované příklady kódu, které by naznačovaly, že tato chyba byla zneužita.

    Pokud chcete ověřit, je-li tato aktualizace zabezpečení v počítači nainstalována, můžete použít nástroj Microsoft Baseline Security Analyzer (MBSA). Další informace naleznete v předchozí části tohoto bulletinu **Nástroje a doporučené postupy zjišťování a nasazení**.

-   **Ověření verze souboru**

    Vzhledem k různým vydáním systému Microsoft Windows se následující postup může lišit od postupu ve vašem systému. Pokud tomu tak je, bude třeba před dokončením následujících kroků konzultovat dokumentaci k produktu.

    1.  Klikněte na tlačítko **Start** a potom na příkaz **Hledat**.
    2.  Ve skupinovém rámečku **Průvodce vyhledáváním** v podokně **Výsledky hledání** klikněte na odkaz **Všechny soubory a složky**.
    3.  Do pole **Část nebo celý název souboru** zadejte název souboru z příslušné tabulky a klikněte na tlačítko **Hledat**.
    4.  V seznamu souborů klikněte pravým tlačítkem myši na název souboru z příslušné tabulky s informacemi o souborech a klikněte na příkaz **Vlastnosti**.  

        **Poznámka** V závislosti na vydání operačního systému nebo nainstalovaných programech nemusejí být některé soubory uvedené v tabulce s informacemi o souborech nainstalovány.
    5.  Na kartě **Verze** zjistěte verzi souboru nainstalovaného v systému a porovnejte ji s verzí popsanou v příslušné tabulce s informacemi o souborech.  

        **Poznámka:** Atributy (kromě verze souboru) se mohou při instalaci změnit. Porovnání jiných atributů souboru s informacemi v příslušné tabulce není spolehlivou metodou ověření instalace aktualizace. V určitých případech mohou být soubory při instalaci přejmenovány. Pokud soubor nebo informace o verzi nejsou k dispozici, použijte k ověření instalace aktualizace některou jinou metodu.
